# Plan: Multi-Theme System

## Goal
Add a theme style selector (dropdown) to the left of "提交issue", supporting "默认主题" and "日本风主题". Each theme has both dark and light variants. The existing dark/light toggle is preserved.

## Architecture Decision

**Keep CSS in `index.html` (no separate files)**. Reasons:
- Project convention: single-file frontend, zero build tools for CSS
- Tauri with `withGlobalTauri` serves from dist; separate CSS files need build pipeline changes
- Only 2 themes for now — separation adds complexity without payoff
- Theme colors are just ~26 variable overrides per combination — manageable inline

**Use compound `data-theme` attribute**: `default`, `default-light`, `japanese`, `japanese-light`
- `:root` → dark defaults (backward compatible)
- `[data-theme="default-light"]` → light overrides (migrated from current `[data-theme="light"]`)
- `[data-theme="japanese"]` → Japanese dark overrides
- `[data-theme="japanese-light"]` → Japanese light overrides

This is the minimum-change approach: the current `[data-theme="light"]` selector just gets renamed.

### Step 5: JS — Add theme style state and functions

New global variables:
```js
var _currentThemeStyle = 'default';  // 'default' | 'japanese'
```

New functions:
- `toggleThemeStyleDrop(e)` — toggle the dropdown open/close
- `selectThemeStyle(style)` — set `_currentThemeStyle`, re-apply theme, save config, close dropdown
- `getThemeDataAttr()` — returns composite theme string: `_currentThemeStyle === 'default' && _currentTheme === 'light' ? 'default-light' : _currentThemeStyle === 'japanese' && _currentTheme === 'light' ? 'japanese-light' : _currentThemeStyle === 'japanese' ? 'japanese' : _currentTheme === 'light' ? 'default-light' : 'default'`

### Step 6: JS — Update existing theme functions

Modify `applyTheme(theme)` to call `getThemeDataAttr()` and set the composite attribute instead of just `'dark'`/`'light'`.

Modify `syncThemeUI()` to also update the theme style dropdown active state.

Modify `syncTitleBarColor(theme)` to handle Japanese theme colors (different title bar RGB for each variant).

Update `collectConfig()` to save `themeStyle: _currentThemeStyle`.

Update `loadAndApplyConfig()` to restore `themeStyle` from config.

Update `syncThemeUI()` to highlight the active option in the style dropdown.

### Step 7: CSS — Add light-theme overrides for Japanese light

Add `[data-theme="japanese-light"]` component overrides (hover states, shadows, ANSI colors), similar to current `[data-theme="light"]` overrides but using the Japanese warm palette.
