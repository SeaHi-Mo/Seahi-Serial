# 修复操作引导定位和聚光灯效果

## 问题分析

从截图看到 3 个问题：

1. **聚光灯无效** — overlay 有 `background: rgba(0,0,0,0.6)` 且卡片有 `box-shadow: 0 0 0 9999px rgba(0,0,0,0.6)`，双重遮盖导致目标元素完全看不到
2. **卡片位置偏移** — 卡片没有紧贴目标元素，而是浮在屏幕中间
3. **箭头方向反了** — CSS 类名 `.top` 把箭头放在卡片底部（指向下），`.bottom` 放在顶部（指向上），与逻辑相反

## 修复方案

修改文件：`src/index.html`

### 1. CSS 修复（第 1541-1575 行）

**overlay**：移除 background，改为纯透明遮罩
```css
#onboarding-overlay {
    position: fixed; inset: 0; z-index: 10000;
    background: transparent;
    pointer-events: none;
}
#onboarding-overlay.show { pointer-events: auto; }
```

**卡片**：移除 9999px box-shadow，保留普通投影
```css
.onboard-step {
    box-shadow: 0 4px 20px rgba(0,0,0,.4);  /* 去掉 0 0 0 9999px */
}
```

**箭头 CSS**：修正方向
- `.onboard-arrow.top` → 箭头在卡片**顶部**（border-bottom + border-right，三角朝上）
- `.onboard-arrow.bottom` → 箭头在卡片**底部**（border-top + border-left，三角朝下）

### 2. JS `goStep()` 重写

**聚光灯**：给目标元素加 `box-shadow: 0 0 0 9999px rgba(0,0,0,0.6)`，切换步骤时先清除上一个

**定位**：
- `pos: 'bottom'` → 卡片在目标下方，`top = rect.bottom + 12`
- `pos: 'top'` → 卡片在目标上方，`top = rect.top - cardHeight - 12`
- `left = rect.left`，左对齐避免溢出
- 边界 clamp

### 3. 步骤调整

| # | 目标 | pos | 说明 |
|---|------|-----|------|
| 1 | `.global-bar` | bottom | 全局操作栏 |
| 2 | `#main-portSelect` | bottom | 端口选择器 |
| 3 | `#main-btnStart` | bottom | 开始按钮 |
| 4 | `#main-sendBar` | top | 发送栏 |
| 5 | `#main-btnAdv` | top | 高级设置 |

### 4. 点击空白关闭

overlay 透明 + pointer-events:auto，点击非卡片区域触发 closeOnboarding()。

## 验证

1. `npm run dev` 启动
2. DevTools: `localStorage.removeItem('onboarding_done'); location.reload();`
3. 检查：卡片紧贴目标、箭头指向正确、目标高亮可见
4. 逐步走完 5 步
5. 点击跳过或空白区域关闭
