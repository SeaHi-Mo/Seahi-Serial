# WSL 监视器同步工作流规则功能

## 现状分析

**额外监视器**：已完全支持工作流，无需修改。`createMonitorPane` 创建时已包含完整的 workflow UI，`startReading` 通过 `read_workflow_events` 显示 `[Auto]` 消息。

**WSL 监视器**：缺少 3 个组件：
1. `monitors['wsl']` 未初始化 `workflows` 属性 → `checkWorkflowMatches` 直接 return
2. WSL HTML 面板缺少 workflow UI 元素（添加按钮、面板容器）
3. 后端 `check_workflow_matches` 已支持任意 monitor_id（含 "wsl"），无需修改

## 不需要修改的部分

| 组件 | 原因 |
|------|------|
| Rust 后端 | `check_workflow_matches`、`save_workflows`、`init_workflows` 已支持任意 monitor_id |
| 额外监视器 | `createMonitorPane` 已包含完整 workflow UI |

## 验证方式

1. `npm run build` 编译
2. 打开 WSL 监视器 → 展开"更多设置" → 应看到"+ 添加规则"按钮
3. 添加规则 → 设置正则匹配条件 → 运行规则
4. WSL 串口收到匹配数据时应触发自动动作
5. 重启应用后 WSL 规则应持久化
