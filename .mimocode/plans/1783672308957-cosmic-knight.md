# 自动重连 Bug 修复：ReconnectController 状态机

## 问题总结

| # | 问题 | 影响 |
|---|------|------|
| C1 | `reconnectPort`/`reconnectWslPort` 的 `setTimeout` 无法取消 | 用户断开后定时器仍触发，端口被意外重新打开 |
| C2 | `_reconnecting` 标志仅在 `setBaud` 中使用，`reconnectPort` 无并发保护 | 波特率切换 + 读取失败同时触发重连，可能损坏后端状态 |
| C3 | `disconnectPort` 不取消重连定时器，且清空 `portName` | 断开后定时器仍运行，`portName` 已空但闭包捕获了旧值 |
| I5 | `setBaud` 重连不检查自动重连开关 | 用户关闭自动重连后波特率切换仍会重试 10 次 |
| I6 | `reconnectPort` 不检查自动重连开关 | 用户关闭自动重连后串口重连仍继续 |

## 方案：ReconnectController 状态机

每个监视器一个控制器实例，存储在 `monitors[mid]._reconnect`，统一管理重连生命周期。

### Controller 设计

```javascript
function ReconnectController(mid, opts) {
  this.mid = mid;
  this.type = opts.type; // 'serial' | 'wsl'
  this._timer = null;
  this._attempt = 0;
  this._portName = '';
  this._active = false;
  this._maxAttempts = 10;
  this._baseDelay = this.type === 'wsl' ? 3000 : 2000;
  this._maxDelay   = this.type === 'wsl' ? 30000 : 20000;
  this._initialDelay = this.type === 'wsl' ? 3000 : 0;
}

ReconnectController.prototype = {
  start: function(portName) {
    if (this._active) return;          // 并发保护
    this._active = true;
    this._attempt = 1;
    this._portName = portName;
    var self = this;
    if (this._initialDelay) {
      this._timer = setTimeout(function() { self._doAttempt(); }, this._initialDelay);
    } else {
      this._doAttempt();
    }
  },

  cancel: function() {                 // 幂等，任何上下文安全调用
    if (this._timer) { clearTimeout(this._timer); this._timer = null; }
    this._active = false;
    this._attempt = 0;
  },

  isActive: function() { return this._active; },

  // ── internal ──
  _doAttempt: function() {
    if (!this._active) return;
    if (!this._portName || this._attempt > this._maxAttempts) {
      this._fail('重连失败，已达到最大重试次数'); return;
    }
    if (!this._checkAutoReconnect()) {
      this._fail('自动重连已关闭'); return;
    }
    var self = this;
    var p = this.type === 'serial' ? this._trySerial() : this._tryWsl();
    p.then(function(ok) { if (!ok) self._scheduleRetry(); });
  },

  _trySerial: async function() {
    var self = this, mid = this.mid;
    await refreshPorts(mid, true);
    var sel = document.getElementById(mid + '-portSelect');
    var found = sel && [...sel.querySelectorAll('.sel-opt')].some(function(o) {
      return o.getAttribute('data-val') === self._portName;
    });
    if (found) {
      // 恢复下拉框到目标端口
      var matchEl = sel.parentElement.querySelector('.sel-drop .sel-opt[data-val="' + this._portName + '"]');
      if (matchEl) {
        sel.querySelector('.sel-text').textContent = matchEl.textContent;
        sel.setAttribute('data-val', this._portName);
        sel.parentElement.querySelectorAll('.sel-opt').forEach(function(o) { o.classList.remove('active'); });
        matchEl.classList.add('active');
      }
      await connectPort(mid, true);
      if (monitors[mid] && monitors[mid].isConnected) { this.cancel(); return true; }
    }
    return false;
  },

  _tryWsl: async function() {
    var mid = this.mid;
    var baud = parseInt(document.getElementById(mid + '-baudRate').value) || 115200;
    try {
      appendOutput(mid, 'sys', '正在重连 (' + this._attempt + '/10): ' + this._portName);
      await invoke('open_wsl_serial', { monitorId: mid, devicePath: this._portName, baudRate: baud });
      monitors[mid].isConnected = true;
      monitors[mid].portName = this._portName;
      updateMonitorUI(mid, true);
      startWslReading(mid);
      appendOutput(mid, 'sys', '重连成功');
      this.cancel();
      return true;
    } catch (e) {
      appendOutput(mid, 'err', '重连失败: ' + e);
      return false;
    }
  },

  _scheduleRetry: function() {
    var self = this;
    var delay = Math.min(this._baseDelay * this._attempt, this._maxDelay);
    appendOutput(this.mid, 'sys',
      '重连失败，' + Math.round(delay / 1000) + '秒后重试 (' + this._attempt + '/' + this._maxAttempts + ')...');
    this._attempt++;
    this._timer = setTimeout(function() { self._doAttempt(); }, delay);
  },

  _checkAutoReconnect: function() {
    var btn = document.getElementById(this.mid + '-btnAutoReconnect');
    return btn && btn.classList.contains('on');
  },

  _fail: function(msg) {
    appendOutput(this.mid, 'err', msg);
    updateMonitorUI(this.mid, false, false);
    this.cancel();
  }
};
```

### 迁移顺序

1. 在 `disconnectPort` 上方（~line 2515）添加 `ReconnectController` 类
2. 在 `createMonitorPane` 和 WSL 初始化中创建控制器实例
3. 更新 `startReading` catch → 调用 `_reconnect.start()`
4. 更新 `startWslReading` catch → 调用 `_reconnect.start()`
5. 更新 `setBaud` → 使用 `isActive()`/`cancel()`/`start()`
6. 在 `disconnectPort`/`disconnectWslPort`/`closeMonitor` 中添加 `cancel()` 调用
7. 删除旧的 `reconnectPort` 和 `reconnectWslPort` 函数

### 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 用户点击断开 | `disconnectPort` 调用 `cancel()` → `clearTimeout` + `_active = false` |
| 用户关闭监视器 | `closeMonitor` 调用 `cancel()` 后 `delete monitors[mid]` |
| 用户关闭自动重连开关 | `_checkAutoReconnect()` 每次重试前检查 → 关闭则 `_fail()` |
| 波特率切换中 | `setBaud` 先 `cancel()`，再尝试连接，失败则 `start()` |
| 端口消失 | `_trySerial` 调用 `refreshPorts` → 端口不在列表 → `_scheduleRetry` |
| 重复调用 `start()` | `if (this._active) return` 守卫 |
| 超过最大重试 | `_doAttempt` 检查 `_attempt > _maxAttempts` → `_fail()` |

## 验证方案

1. **基本重连：** 连接串口 → 拔出设备 → 观察自动重连日志 → 插入设备 → 验证重连成功
2. **取消重连：** 连接串口 → 拔出设备 → 自动重连开始 → 点击断开 → 验证重连停止（无更多日志输出）
3. **开关切换：** 自动重连进行中 → 关闭自动重连按钮 → 验证停止重试
4. **波特率切换：** 连接串口 → 切换波特率 → 验证重连行为正常
5. **监视器关闭：** 自动重连进行中 → 关闭监视器 → 验证无控制台错误
6. **WSL 重连：** WSL 连接 → 断开 → 验证自动重连和取消行为
7. **并发保护：** 快速切换波特率多次 → 验证不会启动多个重连链
