# Fix: Snap Layouts with Custom Title Bar

## Problem
用户需要在自定义标题栏（`decorations: false`）下实现 Windows 11 Snap Layouts（鼠标悬停最大化按钮时出现布局预览）。

## Root Cause
之前失败的原因是 subclass_proc 中返回了 `HTCAPTION`，这告诉 Windows "该区域是标题栏"，导致原生标题栏被渲染出来。同时缺少 `SWP_FRAMECHANGED` 通知系统样式已变化。

## Solution (already implemented in working tree)

### Changes to `src-tauri/src/main.rs`:

1. **Add `enable_snap_layouts` function** (~line 2404):
   - `GetWindowLongW` + `SetWindowLongW` 添加 `WS_MAXIMIZEBOX` 样式位
   - `SetWindowPos` with `SWP_FRAMECHANGED` 通知系统样式变化
   - `SetWindowSubclass` 安装 `WM_NCHITTEST` 拦截器
   - 仅在最大化按钮区域返回 `HTMAXBUTTON`（触发 Snap Layouts）
   - **不返回 `HTCAPTION`** — 窗口拖动由前端 `app-region: drag` 处理
   - 其他消息交给 `DefWindowProcW` 默认处理

2. **在 setup 中调用** `enable_snap_layouts(app)` (line ~2541)

### Key technical points:
- `WS_MAXIMIZEBOX` (0x00010000) 不会导致原生标题栏出现（标题栏由 `WS_CAPTION` 控制）
- `HTMAXBUTTON` 是触发 Snap Layouts 的信号（DWM/shell 检测此命中测试结果）
- `HTCAPTION` 会导致 Windows 渲染原生标题栏（这是之前失败的根因）
- `SWP_FRAMECHANGED` 强制 Windows 发送 `WM_NCCALCSIZE`，确保样式变更生效
- 按钮宽度 `46px` 与前端 CSS `.win-ctrl-btn` 的 `width:46px` 一致
